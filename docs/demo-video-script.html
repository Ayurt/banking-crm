3.1 Intro (0:00–0:45)

On screen: Login → Dashboard

“Hi, I’m Ayush. This is my take-home: a conversation-based Agentic AI for Banking CRM. The problem: a Relationship Manager asks in natural language — find high-value customers likely to convert for a personal loan and generate personalized WhatsApp messages. My system does that end-to-end: retrieve CRM data, score conversion likelihood with explainable rules, recommend products, draft WhatsApp messages, and require human approval before anything is sent.”

3.2 Architecture (0:45–2:00)

On screen: README architecture diagram or packages/ + docs

“Architecture is a monorepo: Frontend React + Vite RM portal; Backend NestJS REST API; Agent core LangGraph multi-agent workflow; Engines for scoring, recommendation, prompts, and evaluation; Data in PostgreSQL with 1,000 seeded customers, Redis, and Chroma configured. Flow: RM query → Planner → parallel retrieval tools → Scoring → Recommendation → Messaging → Audit → Response. Agents share AgentState; they don’t call each other directly. Tools talk to the DB. LLM is used mainly for planning and message generation — scores stay deterministic so they’re explainable.”
Mention:
  • Clear task decomposition (Planner → Retrieval → Score → Rec → Message)
  • Tool/API usage (Customer, Transaction, CRM, Loan, Campaign, Scoring, Recommendation)
  • State/context via LangGraph + conversation persistence
  • Modular packages (extensible)

3.6 Design Decisions & Trade-offs (6:30–8:00)

“Key decisions and trade-offs: 1. Deterministic scoring, LLM for language — numbers stay auditable; LLM personalizes copy. 2. Human approval — banking safety over full automation. 3. LangGraph — shared state, retries, feature flags (memory, audit, streaming). 4. Monorepo packages — scoring/recommendation/prompts can evolve independently. 5. Trade-offs: scoring is heuristic, not trained ML; WhatsApp is draft-only; Chroma is wired for future semantic search; seed is 1k customers for demo scale. 6. Evaluation package — business/security/agent evaluators for quality checks.”

4. Evaluation Checklist — What to Mention

System Design

  • Monorepo: apps/ + packages/ (agent-core, scoring-engine, recommendation-engine, prompts, evaluation-engine)
  • NestJS orchestrator + LangGraph workflow
  • Prisma schema (~16–17 models), seed volumes
  • Feature flags: memory, audit, streaming, cache

Agentic Thinking

  • Natural language → Planner intent + product type
  • Parallel retrieval (customers, transactions, CRM, loans, campaigns)
  • Score → eligibility → recommend → message → audit
  • Shared AgentState, not ad-hoc agent-to-agent calls

Tool Usage
