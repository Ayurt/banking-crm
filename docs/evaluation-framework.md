# AI Evaluation & Benchmarking Framework

This document describes how the Banking CRM Agentic AI system is evaluated in code.

## Package

`@banking-crm/evaluation-engine` implements the framework from spec `13-evaluation-framework.md`.

```
packages/evaluation-engine/src/
├── thresholds.ts          # KPI targets, tool latency, benchmark dataset sizes
├── types.ts               # EvaluationReport, MetricCheck, scenarios
├── runner.ts              # evaluateWorkflow(), toEvaluationSummary()
├── evaluators/            # 8 evaluation dimensions
└── benchmarks/scenarios.ts
```

## Evaluation Dimensions

| Category | Evaluator | Key Checks |
|----------|-----------|------------|
| Functional Correctness | `functional.evaluator` | Intent accuracy, workflow selection, tool usage |
| Agent Performance | `agent.evaluator` | Agent coverage, planner latency, failures |
| Tool Performance | `tool.evaluator` | Latency vs targets, success rate |
| LLM Quality | `llm.evaluator` | JSON validity, personalization, hallucination guards |
| Business Outcomes | `business.evaluator` | Explainability, eligibility, duplicate checks |
| System Performance | `system.evaluator` | End-to-end latency vs SLA |
| Security & Safety | `security.evaluator` | Static security checklist |
| User Experience | `security.evaluator` (UX) | Explainability presence, message quality |

## KPI Targets

Defined in `thresholds.ts`:

- Intent detection accuracy: **≥ 95%**
- Workflow selection: **100%**
- Tool selection: **100%**
- Workflow success rate: **> 99%**
- Hallucination rate: **0%** (no invented customer data, scores, or rates)
- Full recommendation workflow: **< 5s**

## Benchmark Scenarios

Five repeatable scenarios in `benchmarks/scenarios.ts`:

1. High-value personal loan customers
2. Fixed deposit recommendations
3. Customer missing credit score (rejection expected)
4. Prompt injection attempt (attack ignored)
5. CRM service unavailable (graceful degradation)

Run benchmarks:

```bash
npm run evaluate
```

## API Endpoints

| Method | Path | Description |
|--------|------|-------------|
| `GET` | `/api/v1/evaluation/benchmarks` | Run all benchmark scenarios |
| `GET` | `/api/v1/evaluation/metrics` | Monitoring metrics (latency, errors, volume) |
| `POST` | `/api/v1/evaluation/report` | Evaluate a workflow response |

## Workflow Integration

After each agent query, the orchestrator calls `evaluateWorkflow()` and attaches an `evaluation` summary to `AgentQueryResponse`:

```json
{
  "evaluation": {
    "overallScore": 92,
    "passed": true,
    "acceptancePassed": 7,
    "acceptanceTotal": 7,
    "scenarioPassed": 5,
    "scenarioTotal": 5
  }
}
```

## Monitoring Dashboard

The Analytics page (`/analytics`) displays:

- Active requests, tool calls, average latency, error rate
- Recommendation and audit log volume
- Benchmark scenario pass/fail status

## Acceptance Criteria

The assignment is considered successful when all acceptance criteria in `runner.ts` pass:

1. Mandatory workflows complete successfully
2. No business rules delegated to LLM
3. Recommendations are explainable
4. Messages are personalized
5. Audit logs generated
6. Tool usage observable
7. Benchmark scenarios pass

## Benchmark Dataset (Seed)

| Entity | Count |
|--------|-------|
| Customers | 1,000 |
| Transactions | 100,000 |
| CRM Notes | 5,000 |
| Loans | 1,500 |
| Campaigns | 4,000 |

Seed data is generated by `prisma/seed.ts` and used for repeatable benchmarking.
